Senior Developer, Software Architect

Feb 2011 - Jun 2013
Badgeville

Badgeville is a rapidly growing Ruby on Rails tech startup. It provides a SaaS based gamification platform through a RESTful API. The game dynamics (custom rules) customers use can be quite complex. As a result, there can be upward of 100 database queries per API call. The high degree of concurrent requests were causing data corruption due to race conditions. Rapid customer acquisition and a large backlog of product features were constrained by what engineering could provide. Feature development and maintenance were taking longer due to the codebase becoming unwieldy. Additionally, a large portion of the development team were junior developers. These factors limited growth, were cost prohibitive, and created scaling challenges.

Actions
  • Acted as a liason between product and engineering by gathering requirements, architecting solutions, and flushing out specifications to hand off to junior developers.
  • Mentored, performed code reviews, and made myself available as a resource to junior developers.
  • Refactored the core game engine code to use a hybrid DCI pattern.
  • Created an in-memory and inter-request cached data context to act as a go between to the database.
  • Created a fine-grain distributed locking mechanism that allowed requests to still be processed in order and in parallel when there were no data dependencies.
  • Created a prototype for the core game engine that was based on a modified Complex Event Processing (CEP) architecture.
  • Translated hard-coded game mechanics into a simple set of CEP primitives that were easier to work with and more flexible.
  • Architected a solution to generate an abstract syntax tree (AST) from the combined CEP game mechanics to be fed into a compiler stage.
  • Designed an in-memory solution to work in combination with the CEP engine that dramatically reduced required database queries
Results
  • Architecting solutions and handing it off to junior developers allowed them to contribute immediately and quickly get up to speed while still producing a high quality product in a short time frame.
  • The refactor to a DCI pattern reduced maintenance costs by making the core engine code dramatically easier to follow and modify.
  • Eliminated unnecessary duplicate queries and wait for replication delay that resulted in faster responses, reduced database load, and eliminated data corruption caused by replication lag.
  • Eliminated data corruption caused by concurrent requests modifying the same data. 300% faster for create based API calls.
  • CEP based game primitives allowed for greater flexibility while simultaneously simplifying the codebase.
  • CEP prototype is 4000x faster with still more potential once the compilation phase is finished.
